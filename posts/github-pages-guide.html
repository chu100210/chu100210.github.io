GitHub Pages 搭建指南

2026-07-15  ·  教程

GitHub Pages 是 GitHub 提供的免费静态网站托管服务。只要把 HTML 文件推到仓库，它就自动变成一个网站。这篇文章记录从零开始搭建的全过程。

第一步：创建仓库

在 GitHub 上新建一个仓库，仓库名格式为 你的用户名.github.io。比如你的 GitHub 用户名是 zhangsan，仓库名就填 zhangsan.github.io。

仓库设为 Public（公开），这是 GitHub Pages 免费版的要求。

第二步：把代码推上去

git init
git add .
git commit -m "init blog"
git branch -M main
git remote add origin git@github.com:你的用户名/你的用户名.github.io.git
git push -u origin main

推送成功后，等一两分钟，访问 https://你的用户名.github.io 就能看到你的网站了。

第三步：开启 GitHub Pages（如果需要）

去仓库的 Settings → Pages，确认 Source 选的是 Deploy from a branch，分支选 main，目录选 / (root)。一般情况下推送后会自动配置好。

绑定自定义域名（可选）

如果你有自己的域名，可以绑定到 GitHub Pages：

  1. 在仓库根目录创建 CNAME 文件，里面只写一行你的域名
  2. 去域名的 DNS 后台，添加一条 CNAME 记录，指向 你的用户名.github.io
  3. 在 Settings → Pages 里勾选 Enforce HTTPS

搞定之后，你的域名就指向了 GitHub Pages，而且自带 HTTPS 证书。

总结

GitHub Pages 的优点很明显：

  • 完全免费，没有流量限制（合理使用范围内）
  • 自动 HTTPS
  • 推送即部署，没有额外步骤
  • 可以绑定自定义域名

缺点是只能托管静态内容（HTML/CSS/JS），不能跑后端。不过对个人博客来说，静态站完全够用了。
